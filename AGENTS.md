# AGENTS.md

Этот файл задаёт правила для ИИ-агентов, которые развивают репозиторий `site-control-kit`.

## Миссия
Поддерживать и улучшать `Site Control Kit` как локальную платформу управления сайтами:
- Python-хаб команд + очередь + CLI;
- браузерное расширение-мост + content script.

## Неприкосновенные правила
1. По возможности сохранять обратную совместимость API.
2. Никогда не ослаблять проверку токена/авторизации без явного решения.
3. Любые изменения протокола документировать в `docs/API.md`.
4. Для несовместимых изменений добавлять заметки миграции.
5. Не добавлять внешние зависимости без технического обоснования.

## Источники правды
- Протокол: `docs/API.md`
- Архитектура: `docs/ARCHITECTURE.md`
- Безопасность: `docs/SECURITY.md`

## Минимальные проверки
- Обязательно запускать: `python3 -m unittest discover -s tests -p 'test_*.py'`.
- При изменении протокола обновлять `examples/`.
- При изменении команд расширения обновлять матрицу команд в `docs/EXTENSION.md`.

## Стандарты кода
- Python: приоритет стандартной библиотеки, явная валидация, понятные ошибки.
- JS (extension): без лишнего tooling, читабельные plain-файлы.
- Комментарии: только для неочевидной логики и security-нюансов.

## Операционные замечания
- Service worker MV3 может «засыпать», polling должен быть устойчивым (таймеры + alarms).
- Поддерживать корректную маршрутизацию по `client_id`.
- Состояние в `state.json` должно оставаться детерминированным и удобным для ручной диагностики.
