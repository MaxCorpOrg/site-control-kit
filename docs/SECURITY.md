# Безопасность

## Аутентификация
- Хаб принимает общий токен.
- Токен должен совпадать у хаба, CLI и расширения.

Быстрый локальный режим:
- токен по умолчанию `local-bridge-quickstart-2026`
- использовать только на личной локальной машине.

Для безопасного режима:
- обязательно задайте свой сильный токен (`SITECTL_TOKEN`).

## Риски
- Если токен утечёт, локальные команды можно отправлять от чужого процесса.
- Если открыть хаб наружу без защиты, возможен несанкционированный доступ.
- `run_script` может выполнять произвольную логику в контексте страницы.

## Рекомендации
1. Держать хаб на `127.0.0.1`.
2. Использовать длинный случайный токен.
3. Регулярно менять токен.
4. Не публиковать хаб напрямую в интернет.
5. Если нужен удалённый доступ — только через защищённый reverse proxy + auth + TLS.
6. Не хранить чувствительные секреты в командах без необходимости.

## Данные и приватность
- `state.json` содержит историю команд и результаты.
- Для чувствительных сценариев регулярно очищайте state-файл.

## Правовое использование
Используйте только там, где у вас есть право автоматизировать действия.
