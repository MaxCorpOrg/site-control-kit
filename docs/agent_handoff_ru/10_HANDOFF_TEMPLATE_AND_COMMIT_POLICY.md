# Handoff Template And Commit Policy

## Как Оставлять Handoff После Задачи
После каждого заметного блока агент должен обновить:
- `AGENT_START_HERE.md`
- `CODEX_STATE.md`
- `docs/PROJECT_STATUS_RU.md`
- при необходимости `README.md`
- если меняется порядок работы агентов, `AGENTS.md` и/или `docs/PROJECT_WORKFLOW_RU.md`

Это правило действует не только для больших этапов.
Даже после маленькой завершённой задачи агент обязан оставить короткую handoff-заметку.

Если после задачи планируется `commit` или `push`, сначала нужно:
- сохранить текущую контрольную точку в handoff-файлах;
- явно записать, что проверено live/тестами;
- оставить следующий шаг, с которого можно продолжить новый разговор без повторного расследования.

## Минимальный Handoff Формат
Нужно оставить пять вещей:
1. Что сделано.
2. Что проверено.
3. Где лежат live artifacts.
4. Что осталось проблемой.
5. Что делать следующим шагом.

Если live artifacts нет, вместо них всё равно нужно указать:
- какие файлы были изменены;
- где теперь смотреть следующему агенту.

## Рекомендуемый Текст Handoff
Пример структуры:
- Сделано: ...
- Проверено: ...
- Артефакты: ...
- Текущий риск: ...
- Следующий приоритет: ...

## Как Писать Коммиты
- писать по-русски;
- один коммит = один логический блок;
- не смешивать unrelated refactor и runtime fix без необходимости.

## Хорошие Примеры Названий Коммитов
- `Ускорил chain-runner после productive deep yield`
- `Добавил профили fast balanced deep для chain-runner`
- `Вынес Telegram профили в общий helper и починил forced tab`

## Что Нужно Записать В Commit-Связке
Если есть live smoke, в handoff-документе обязательно должны остаться:
- путь к `run.json`
- путь к `export.log`
- путь к `export_stats.json`, если он был

Если работа шла по текущему Telegram GUI `tdata-history-authors` path, рядом нужно оставить ещё и:
- был ли это полный history-run без timeout или частичный/stop smoke;
- каким был progress/status UX;
- где лежат `.md` и `*_usernames.*`, если они уже построены.

## Когда Нельзя Считать Работу Завершённой
Нельзя считать задачу законченной, если:
- не прогнаны тесты;
- не обновлён status/handoff;
- нет ясного next step;
- агент не может объяснить, где именно теперь узкий момент.
- не оставлены заметки о том, что сделано и что нужно сделать дальше.

## Как Использовать Этот Пакет В Новом Чате
В новом чате достаточно написать, что агент должен начать с:
- `docs/agent_handoff_ru/00_START_HERE.md`
- потом пройти по всему пакету по номерам

Это даёт будущему агенту предсказуемый onboarding без восстановления контекста по кускам.
