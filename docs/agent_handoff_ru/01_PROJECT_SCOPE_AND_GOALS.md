# Project Scope And Goals

## Что Это За Проект
`site-control-kit` — это локальная платформа управления браузером.
Проект состоит из Python-хаба, CLI, MV3-расширения и прикладных сценариев поверх них.

Сейчас самая развитая прикладная часть — Telegram export pipeline, который собирает usernames из Telegram Web и сохраняет их в безопасной структуре артефактов.

## Основные Цели Проекта
1. Дать агенту и оператору воспроизводимый локальный browser bridge.
2. Поддерживать queue/result модель через hub как единый источник истины.
3. Иметь стабильный CLI-слой для живых browser-команд.
4. Довести Telegram export до максимально автоматического, безопасного и проверяемого контура.

## Что Считается Успехом
Проект успешен, если:
- живой browser client стабильно виден в hub;
- agent может выбирать client/tab предсказуемо;
- DOM-команды расширения реально исполняются в вкладках;
- Telegram pipeline собирает реальные `@username`;
- спорные результаты не попадают в safe-слой;
- каждый run оставляет понятные диагностические артефакты.

## Что Не Является Целью Прямо Сейчас
- универсальный Selenium replacement для всего браузера;
- облачный сервис;
- многопользовательская серверная система;
- идеальная UI-полировка поверх shell/zenity;
- полный уход от Telegram DOM в пользу официального API.

## Ключевая Практическая Миссия Сейчас
Текущая миссия Telegram-направления:
- собирать `@username` из живого чата Telegram Web;
- сохранять их партиями в `~/telegram_contact_batches/chat_<id>/N.txt`;
- разделять raw и safe слои;
- не портить полезные historical snapshots слабым run;
- постепенно повышать throughput deep-path.

## Что Уже Достигнуто
- есть working chain-runner;
- есть profile presets `fast / balanced / deep`;
- есть history backfill;
- есть safe snapshots;
- есть review/conflicts quarantine;
- есть discovery state между прогонами;
- есть live-подтверждённый mention/url fallback path.

## Главная Нынешняя Граница
Граница проекта сейчас уже не в том, чтобы "вообще что-то собрать".
Граница теперь в том, чтобы:
- собирать больше новых реальных `@username` за единицу времени;
- уменьшать потери времени на flaky `click_menu_text`;
- удерживать корректный возврат в group dialog;
- не терять управление вкладкой и tab targeting.
